# Программа для работы с банковскими операциями

## Описание:
### Программа содержит функции:
- для маскировки номера карты или банковского счёта
- для сортировки банковских операций по их статусу
- для сортировки всех банковских операций по дате их выполнения
- для сортировки транзакций по ключу 'state'
- для сортировки транзакций по дате в зависимости от значения параметра increase
- для создания итератора, который поочередно выдает транзакции с заданной валютой
- для определения описания каждой транзакции по очереди
- для генерации номера карты в диапазоне от 0000 0000 0000 0001 до 
    9999 9999 9999 9999, в зависимости от начального и 
    конечного значения номеров
- для логирования функций
- для чтения JSON-, CSV- или Excel-файлов
- конвертации транзакций в рубли по текущему курсу валют 
  с помощью Exchange Rates Data API
- для осуществления поиска в списке словарей операций по заданной строке
  с использованием библиотеки re
- для подсчёта количества банковских операций в списке

## Использование:

#### Работа основной программы
##### Описание:
    Программа приветствует пользователя.

    Программа предлагает выбрать файл, из которого нужно
    получить информацию о транзакциях: 

    "1. Получить информацию о транзакциях из JSON-файла
    2. Получить информацию о транзакциях из CSV-файла
    3. Получить информацию о транзакциях из XLSX-файла"

    После пользователь выбирает статус интересующих его операций:

    "Введите статус, по которому необходимо выполнить фильтрацию. 
    Доступные для фильтровки статусы: EXECUTED, CANCELED, PENDING"

    В случае, если Пользователь ввел неверный статус, программа 
    возвращать Пользователя к вводу корректного статуса.

    После фильтрации программа выводит следующие вопросы для 
    уточнения выборки операций, необходимых пользователю: 

    "Отсортировать операции по дате? Да/Нет"

    "Отсортировать по возрастанию или по убыванию? 
    (по возрастанию/по убыванию)"

    "Выводить только рублевые тразакции? Да/Нет"

    "Отфильтровать список транзакций по определенному слову 
    в описании? Да/Нет"

    Если выбран ответ "Да", то программа выдаст запрос:
    
    "Введите слово для поиска"

9. #### Декоратор log 
##### Описание:
    автоматически логирует начало и конец выполнения функции, а 
    также ее результаты или возникшие ошибки. Декоратор принимает
    необязательный аргумент filename, который определяет, куда 
    будут записываться логи (в файл или в консоль).
    Логирование должно включает:
    - Имя функции и результат выполнения при успешной операции.
    - Имя функции, тип возникшей ошибки и входные параметры, если
      выполнение функции привело к ошибке. 
##### Пример использования функции:

```commandline
python
@log(filename="function_log")
def summ_two_numbers(x: int | float, y: int | float) -> int | float:
    """
    Складывает два числа
    """
    return x + y
    
summ_two_numbers(1, 2)
summ_two_numbers(1, "2")
```

11. #### Функция get_amount_of_transaction
##### Описание:
    принимает на вход транзакцию и возвращает сумму транзакции (amount)
    в рублях, тип данных — float. Если транзакция была в USD или EUR,

    и конвертации суммы операции в рубли. Для конвертации валюты 
    используется Exchange Rates Data API: 
    https://apilayer.com/exchangerates_data-api. 
    Функцию конвертации расположена в модуле external_api.
##### Пример использования функции:

15. ### Функция count_operations_in_categories
##### Описание:
    Считает количество банковских операций определенного типа.
    Принимает два аргумента: список с транзакциями и словарь 
    для подсчета транзакций по описанию.Функция использует 
    Counter из библиотеки collections. Возвращает словарь, в
    котором ключи — это названия категорий, а значения — это
    количество операций в каждой категории.
##### Пример использования

```commandline
# Пример вызова функции:
```
